# NilBot Tabletop v0.1 — everything so far

**Branch:** main · **Date:** 2026-07-22 · **Author:** Denis (+ Claude)

The complete record of the Tabletop, from first Electron shell to the
Portal workspace. Granular history lives in git (`bb3ac90..64023f5`);
this is the map.

## The shell (electron/)
- **Electron app** boots the SvelteKit adapter-node build with its data
  in the OS user-data folder; dev runs on system node, packaged builds
  will use utilityProcess (electron-builder.yml staged, unused yet).
  The server binds the LAN so players' phones/TV still reach Battle
  Ready and the canvas. Single instance; external links open in the
  real browser; close = quit.
- **Campaigns are save slots.** Each campaign is a fully separate world
  (own database + files) under `userData/campaigns/<id>/`. The shell
  silently creates and signs into a per-campaign "Dungeon Master"
  account over HTTP — **the game has no login screen**. Create/open/
  remove (remove keeps files) from the Campaigns screen; opening swaps
  the running server.
- **Boot sequence:** "THE GIANT SLAYERS presents" studio card (pixel
  crossed-swords emblem, near-static after two smoothness passes —
  fade in, hold, cut; any key skips) → title screen.
- **Title screen:** pixel-art night camp, procedurally drawn at 192×108
  and upscaled — dithered navy sky, twinkling stars, pixel moon, night
  clouds, layered hills over a moonlit meadow, fireflies, and a
  hand-drawn sprite campfire (stone ring, crossed logs, 3-frame flame,
  smoke, ground glow) with a sword planted beside it. Menu: Campaigns /
  Join a Campaign (stub for the squid bridge) / Settings / Quit, VT323
  with ▶ hover selector. (Earlier directions — CSS gradients, a
  Bierstadt painting, bright overworld, day camp — were auditioned and
  rejected on the way here.)
- **Shell settings:** fullscreen toggle persisted in prefs.json
  (restores on boot), campaign-files folder opener, version display.
- **Fonts bundled** in electron/fonts: VT323 (primary pixel type),
  Jacquard 12, Cinzel (legacy).
- **Dev launcher:** .desktop entry ("NilBot Tabletop", goblin icon,
  Games category) in the app menu + desktop, with nvm-safe PATH.

## The fork
- History-preserving fork of NilBot (upstream remote kept — merge core
  improvements with `git fetch upstream && git merge upstream/main`).
- The interim injection layer (insertCSS/executeJavaScript theming
  from the shell) was fought and killed: SvelteKit hydration stomped
  injected DOM. Everything moved into `src/` as first-class code; the
  shell now contains zero injection calls.

## The app (src/ — freely divergent from web NilBot)
- **Arcane sanctum theme:** deep-navy palette, mana-cyan accent, VT323
  headings with hard shadows, square corners everywhere, chunky
  borders, pixel scrollbars.
- **ManaField** (`src/lib/components/ManaField.svelte`): living pixel
  backdrop behind the whole app — depth gradient, faint stars, drifting
  glowing motes — with **section moods**: Combat wings burn (fire
  embers, no stars, torch-orange accents), Library charges Grimoire
  violet, World settles Emporium gold, the Portal glows squid purple,
  everything else stays sanctum cyan. Accents follow via
  `.shell[data-room]` CSS variables.
- **Sidebar in pixel type with true names:** Chronicles (Sessions),
  The Table (Battle Ready), Battlegrounds (Battle Maps), The Forge
  (Sheet Builder), The Atlas (World Maps), The Bard (Music), The
  Namesmith (Name Generator), The Emporium (Shop), The Grimoire
  (Spells); page headers match. Bestiary/Battles/Characters/Journal/
  Settings kept their names. Native ◀ MENU button returns to the title
  screen (shell intercepts /__title).
- **Dashboard:** "Well met, {DM}" + rotating flavor lines.
- **In-world Settings, de-webbed:** logout removed (leaving a world is
  ◀ MENU), change-password removed (would desync the shell's stored
  credential and lock the world), storage quota gauge removed and the
  cap lifted via shell env (local worlds are bounded by the disk).

## The Portal (new, Tabletop-only)
- **Own workspace** at /portal with its own sidebar in squid purple —
  the DM's end of the future player bridge. Rooms:
  - **Overview:** party roster management (add/remove players with
    portraits) + **Game Nights** scheduling (new `schedule_events`
    table + /api/schedule): title/date/note, upcoming with countdowns,
    the recent past lingers 24h.
  - **Players:** the interactable page — per-player cards with a gold
    purse (grant/dock arbitrary amounts, floored at 0) and 5e condition
    chips (afflict from a picker, click to lift). New `pcs.gold` and
    `pcs.conditions` columns; PATCH /api/portal/players/[id]. Linked
    sheet shortcut per player.
  - **The Scrying Pool:** declared receiving surface for player
    actions ("X casts Firebolt" → bless/veto, slots counted) once the
    squid-side bridge exists.

## How to test
`npm run desktop` (first time: `npm i`, approve/rebuild better-sqlite3;
Linux sandbox note in CLAUDE.md). Boot card → title → create a world →
sanctum. Walk the wings for the moods, open the Portal, grant gold,
inflict "poisoned", schedule a game night.

## Update — invites forge the sheet; the seat becomes a dashboard
- **Cutting a key forges the character.** The Overview invite form is now
  Player name + Character name + Class; the API creates the `pcs` row on
  the spot — level 1, class-appropriate standard-array stats, SRD starting
  gold and gear kit (`src/lib/classnotes.ts` is the class ledger:
  hit die, primary, saves, caster kind, kit, blurb, default stats).
  A PC name that already exists in the party binds instead of duplicating.
- **Schema:** `pcs` gains `level, str, dex, con, intel, wis, cha`
  (defaults 1/10) and `items` (JSON array text) — idempotent ALTERs after
  the `pcs` CREATE.
- **Portal → Players** now manages the whole sheet: level stepper (1–20),
  six editable ability scores, item bestow/take-away, alongside the
  existing gold and conditions. PATCH accepts `level`,
  `stat {key,value}` (whitelisted column), `addItem`, `removeItem`.
- **The player seat is a real dashboard** (`/table` reshaped into a
  layout + pages, seat resolution shared via `src/lib/server/seat.ts`):
  - Sidebar shell: character name, class & level, class lore box
    (hit die / primary / saves / blurb), nav — My Sheet, Spells
    (casters only), The Party — and the MENU button.
  - **My Sheet** (`/table`, where an accepted invite lands): stat tiles
    with modifiers, gold, conditions, items — live, DM-managed.
  - **Spells** (`/table/spells`): the compendium filtered to the
    character's class and castable levels (full/pact: ceil(lvl/2) max 9;
    half-casters from level 2, ceil(lvl/4) max 5), grouped by level,
    searchable, expandable full text. Empty-compendium and non-caster
    states explained in place.
  - **The Party** (`/table/party`): roster with levels + next game night
    (moved off the old single seat page).

### Deploy steps
None — existing campaign worlds pick up the new pcs columns on next boot.
Spells for players require the world's compendium to be seeded
(`node scripts/import-open5e-spells.mjs` inside the campaign world dir) —
still the top known gap.

## Update — worlds get stocked; items carry their lore
- **`scripts/seed-world.mjs`** pours an existing NilBot data dir into a
  campaign world: shared compendium (monsters upserted by slug + instance
  token art into `data/tokens/`, items, spells) and all maps (rows become
  the world DM's, files copied into the world store, storage accounted).
  Idempotent — rerun after new imports. Operator-run on your own machine
  only (licensing rule 1 — never bake seeded worlds into a distributed
  image). Usage:
  `node scripts/seed-world.mjs --from ../NilBot/data --world ~/.config/nilbot-tabletop/campaigns/<id>`
- **Granting items now speaks the catalog**: the character page's give box
  suggests from the world's item catalog; a granted item that matches it
  arrives on the player's sheet expandable — type · rarity, attunement,
  full description. Free-text items still work (they just carry no lore).
- **seed-world also carries the personal layer** (skip with
  `--no-personal`, source account via `--from-user <id>`, default 1):
  characters with portraits + groups, party PCs, notes, quick notes,
  journal pages (parent links preserved), battles (note/map/PC-token
  references remapped to the world's new ids), songs, shop stock. All
  idempotent by natural keys.
